# HelixQA v3.0 — OpenClawing2 Integration

**Release date:** 2026-04-18
**Plan:** `docs/plans/2026-04-18-openclawing2-integration-plan.md`
**Research source:** `docs/OpenClawing2.md`

HelixQA v3.0 ports every high-priority pattern from the five
reference frameworks the OpenClawing2 analysis evaluated, while
preserving HelixQA's project-agnostic posture and the
Apache-2.0 licence boundary.

---

## New packages + capabilities

### `pkg/nexus/agent` — Four-phase state machine

Ports browser-use's `Agent.step()` loop (see
`tools/opensource/browser-use/browser_use/agent/service.py`).

- `AgentStep` typed output (`Evaluation`, `Memory`, `NextGoal`,
  `Actions`, `Done`) + `AgentState` history.
- Explicit phases: `PrepareContext → PlanActions → Execute →
  PostProcess`.
- `Agent.Run()` with `ErrMaxIterationsExceeded` safety net,
  `DefaultSystemPrompt` vendored, `ParsePlannerJSON` helper.
- Structured integration tests prove Phases 2–5 compose.

### `pkg/nexus/agent` — MessageManager (Phase 3)

- `PrepareStepState` + `CreateStateMessages` + `Compact` +
  `Messages` + `TotalTokens` / `Digest` / `WindowSize`.
- Digest folds accumulated summary + new batch into a single
  summariser pass so the prompt never grows unbounded.
- `DefaultSummariser` caps output at 512 bytes for offline-safe
  tests.
- 500-step stress session stays inside configurable token budget.

### `pkg/nexus/agent` — Retry / LoopDetector / SelfHealer (Phase 4)

- `BackoffPolicy` with safe defaults + per-attempt jitter.
- `RetryWithBackoff` honours `ctx.Done()` mid-sleep.
- `LoopDetector` catches 2- and 3-cycles via FNV fingerprints.
- `SelfHealer` re-invokes the planner with an explicit
  `previous_attempt_failed_because:` hint prepended to
  `PlanRequest.RecentSteps`.

### `pkg/nexus/primitives` — Stagehand primitives (Phase 5)

- `Engine.Act(ctx, nl)` — NL → single `nexus.Action`.
- `Engine.Extract(ctx, prompt, schema, out)` — typed extraction.
- `Engine.Observe(ctx, descriptor)` — descriptor → refs.
- `Engine.Agent(ctx, goal, cfg)` — scoped autonomous mode.
- `PromptCache` keyed on snapshot + prompt + schema SHA-256.
- `WithSelfHealer` adds re-inference on dispatch failure.

### `pkg/nexus/coordinate` — Coord-grounded actions (Phase 6)

- `ScaleCoordinates` implements Anthropic's `scale_coordinates`
  algorithm with the `MAX_SCALING_TARGETS` catalogue.
- Normalised (0..1) UI-TARS-style inputs supported.
- `CoordClick` / `CoordType` / `CoordDrag` / `CoordScroll`
  builders emit ready-to-dispatch `nexus.Action`s with the
  canonical kind constants.
- `ErrOutOfBounds` stops rogue clicks from escaping the screen.

### `pkg/ticket` — Rich evidence capture (Phase 7)

- `Ticket` gains seven optional fields: `VideoTimestamp`,
  `BeforeScreenshotPath`, `AfterScreenshotPath`,
  `LLMReasoningTranscript`, `ReproductionBank`, `SessionID`,
  `StepNumber`.
- `CaptureRich(t, input)` merges evidence; skips zero fields so
  legacy callers upgrade gracefully.
- `RenderRichMarkdown` emits a reviewer-friendly `## Evidence`
  block.
- `VerifyMandatory` + `MandatoryRichFields` gate incomplete
  tickets.

### `pkg/opensource` — Vendored-submodule audit (Phase 1)

- Every subdirectory of `tools/opensource/` is documented in
  `docs/opensource-references.md` and licensed in
  `docs/licences-inventory.md`.
- `pkg/opensource.Audit` + `LocateLicenceFile` enforce the gate
  at PR time; 4 regression tests lock it in.
- `scripts/sync-opensource-references.sh` dry-runs SHA diffs
  before operators bump the pins.

## Operational notes

- **No new external runtime dependencies.** Every ported pattern is
  native Go or uses a module already in `go.mod`.
- **Project-agnostic.** Re-audit with
  `banks/fixes-validation-decoupling.yaml::FIX-DECOUPLE-006` —
  no Catalogizer-specific strings land in production code.
- **Rootless.** Campaign script never calls `sudo`.
- **Offline-first.** Tests do not hit network; `DefaultSummariser`
  is pure string manipulation.
- **Prompt cache never persists to disk.** SHA-256 keys live in
  memory; default 5-minute TTL; opt-out via
  `WithPromptCache(nil)`.
